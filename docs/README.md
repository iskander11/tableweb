# TableWeb — Документация

Корпоративное веб-приложение для совместной работы с Excel-таблицами прямо в браузере.

---

## Что умеет программа (простыми словами)

- **Открывать таблицы в браузере** — как Excel, но через интернет, без установки программ
- **Несколько человек могут работать одновременно** — изменения одного видны всем в реальном времени
- **Импортировать Excel-файлы** — загрузить `.xlsx` и он откроется с сохранением цветов, размеров, формул
- **Экспортировать обратно в Excel** — скачать таблицу в `.xlsx`
- **Управлять пользователями** — кто может только смотреть, кто может редактировать
- **История изменений** — видно, кто и когда сохранял документ
- **Резервные копии** — на случай если что-то пошло не так

---

## Роли пользователей

| Роль | Что может делать |
|------|-----------------|
| **Администратор** | Всё: создавать/удалять таблицы, управлять пользователями, делать бэкапы |
| **Редактор** | Создавать таблицы, редактировать, импортировать/экспортировать Excel |
| **Читатель** | Только просматривать таблицы (редактирование недоступно) |

## Как пользоваться — инструкция

### Вход в систему
1. Откройте браузер → введите адрес сервера (например `http://<YOUR_SERVER_IP>`)
2. Введите логин и пароль → нажмите **Войти**

### Создание таблицы (редактор или администратор)
1. На главной странице нажмите **+ Новая таблица**
2. Введите название → нажмите **Создать**
3. Таблица откроется — можно сразу начинать вводить данные

### Редактирование таблицы
1. Нажмите на таблицу в списке — она откроется
2. Кликните на нужную ячейку и введите данные
3. **ВАЖНО**: нажмите кнопку **«Сохранить»** в верхнем правом углу, чтобы данные сохранились
   - Кнопка светится оранжевым — есть несохранённые изменения
   - Кнопка стала зелёной «Сохранено» — всё ОК

> **Если не нажать Сохранить** и закрыть страницу — изменения будут потеряны!

### Режим чтения (читатель)
- При открытии таблицы вверху появится синяя плашка: **«Вы в режиме чтения — редактирование недоступно»**
- Панель инструментов не отображается
- Можно скачать таблицу (кнопка Экспорт)

### История изменений
1. В верхнем углу страницы таблицы нажмите кнопку **«История»** (иконка часов)
2. Справа откроется панель — там показано кто и когда сохранял документ
3. Нажмите **«История»** ещё раз чтобы закрыть панель

### Импорт Excel-файла
1. Откройте таблицу
2. Нажмите **Импорт** → выберите `.xlsx` или `.xls` файл
3. Дождитесь окончания импорта (показывается прогресс)
4. Данные загрузятся — форматирование, цвета, размеры колонок сохраняются

### Экспорт в Excel
1. Откройте таблицу
2. Нажмите **Экспорт** — файл скачается в браузер

### Совместная работа
- Когда несколько человек открывают одну таблицу — в шапке видно их имена
- Изменения одного пользователя мгновенно видны другому (real-time)
- Каждый нажимает Сохранить самостоятельно

## Управление пользователями (только администратор)

### Создать нового пользователя
1. Войдите как администратор
2. Нажмите кнопку **Настройки** (шестерёнка в шапке)
3. Раздел «Создать пользователя»
4. Введите логин, пароль, выберите роль → нажмите **Создать**

**Правила:**
- Логин: от 2 до 50 символов, уникальный
- Пароль: минимум 4 символа
- Если пользователь был ранее деактивирован — он будет восстановлен с новым паролем

### Изменить роль пользователя
В списке пользователей → рядом с именем выберите роль из выпадающего списка

### Деактивировать (удалить) пользователя
В списке пользователей → нажмите на красную иконку корзины

Пользователь НЕ удаляется из базы полностью, а только деактивируется (soft delete).
Он появится в разделе «Деактивированные» и его можно восстановить.

### Восстановить пользователя
В разделе «Деактивированные» → нажмите кнопку **Восстановить**

### Передать права администратора
1. Настройки → раздел «Передать права администратора»
2. Выберите пользователя → нажмите **Передать**
3. Вы автоматически становитесь читателем и выходите из системы

## Резервные копии (только администратор)

Система поддерживает три типа резервных копий:

| Тип | Значок | Когда | Охват |
|-----|--------|-------|-------|
| **Недельный** (синий) | Автоматически | Каждое воскресенье в 02:00 | Все таблицы в системе |
| **Дневной** (жёлтый) | Вручную, иконка 📦 у таблицы | По запросу | Одна выбранная таблица |
| **Ручной** (серый) | Кнопка «Создать бэкап» в Настройках | По запросу | Все таблицы |

### Создать дневной бэкап одной таблицы
На главной странице → нажмите иконку архива справа от нужной таблицы.  
Иконка на 3 секунды зеленеет — бэкап создан.

### Создать полный бэкап вручную
1. Настройки → раздел «Резервные копии»
2. Нажмите **Создать бэкап** — ZIP со всеми таблицами

### Когда будет следующий автобэкап
В разделе «Резервные копии» (Настройки) отображается дата и время следующего автоматического недельного бэкапа.

### Скачать бэкап
В списке бэкапов → нажмите иконку скачивания

### Восстановить из бэкапа
В списке бэкапов → нажмите **Восстановить**

Таблицы создадутся заново (если такая таблица уже есть — создаётся с суффиксом, оригинал не перезаписывается).

## Пользовательские шрифты (только администратор)

Если таблица отображается не так как в Excel — нужно загрузить нужный шрифт.

1. Настройки → раздел «Шрифты»
2. Нажмите «Выбрать файл» → выберите `.ttf`, `.otf`, `.woff` или `.woff2`
3. Имя шрифта определяется автоматически (можно задать вручную)
4. Нажмите **Загрузить**

После загрузки шрифт начнёт использоваться в таблицах автоматически.

## Техническая информация

### Стек технологий

| Слой | Технология |
|------|-----------|
| Frontend | React 18 + Vite + TypeScript + TailwindCSS |
| Excel-движок | FortuneSheet (open-source) |
| Excel I/O | Python-микросервис (Flask + openpyxl) |
| Backend | Node.js + Express + Socket.io |
| База данных | PostgreSQL |
| Real-time | WebSockets (Socket.io) |
| Деплой | Nginx + PM2 |

### Структура файлов

```
tableweb/
├── app/
│   ├── frontend/          # React приложение
│   │   └── src/
│   │       ├── pages/     # LoginPage, DashboardPage, SheetPage, AdminPage
│   │       ├── store/     # Zustand (auth state)
│   │       ├── fonts/     # Загрузка шрифтов
│   │       └── api/       # axios client
│   ├── backend/           # Node.js API
│   │   └── src/
│   │       ├── routes/    # auth, spreadsheets, excel, backup, fonts
│   │       ├── services/  # excel.js (проксирует в excel-service)
│   │       ├── middleware/ # auth.js (JWT + роли)
│   │       └── db/        # PostgreSQL pool + schema.sql
│   └── excel-service/     # Python Flask (парсинг .xlsx)
├── deploy/
│   ├── setup-vps.sh       # Первоначальная установка на сервер
│   └── update.sh          # Обновление кода на сервере
├── vps/
│   ├── id_rsa             # SSH ключ для подключения к серверу
│   └── ip.txt             # IP адрес сервера
└── docs/
    └── README.md          # Эта документация
```

### База данных — таблицы

| Таблица | Что хранит |
|---------|-----------|
| `users` | Пользователи (логин, хэш пароля, роль, активен/нет) |
| `spreadsheets` | Метаданные таблиц (название, кто создал) |
| `spreadsheet_data` | Данные таблиц в формате JSON (ячейки, стили) |
| `spreadsheet_permissions` | Кому и с какой ролью доступна таблица |
| `change_log` | История изменений: кто, когда, какой лист, какие ячейки сохранил |
| `backups` | Метаданные резервных копий (тип: weekly/daily/manual, название таблицы) |
| `fonts` | Загруженные шрифты |
| `admin_transfers` | Лог передачи прав администратора |

## API Endpoints (справочник для разработчиков)

### Аутентификация
| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/api/auth/login` | Вход в систему |
| GET | `/api/auth/me` | Текущий пользователь |
| POST | `/api/auth/users` | Создать пользователя (admin) |
| GET | `/api/auth/users` | Список пользователей (admin) |
| PATCH | `/api/auth/users/:id/role` | Изменить роль (admin) |
| PATCH | `/api/auth/users/:id/reactivate` | Восстановить пользователя (admin) |
| DELETE | `/api/auth/users/:id` | Деактивировать пользователя (admin) |
| POST | `/api/auth/transfer-admin` | Передать права администратора |
| PATCH | `/api/auth/password` | Сменить собственный пароль |
| DELETE | `/api/auth/users/:id/permanent` | Удалить пользователя навсегда (admin, только неактивных) |
| PATCH | `/api/auth/users/:id/reactivate` | Восстановить деактивированного пользователя (admin) |

### Таблицы
| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/api/spreadsheets` | Список доступных таблиц |
| POST | `/api/spreadsheets` | Создать таблицу |
| GET | `/api/spreadsheets/:id` | Данные таблицы |
| GET | `/api/spreadsheets/:id/changelog` | История изменений таблицы |
| PATCH | `/api/spreadsheets/:id/rename` | Переименовать |
| DELETE | `/api/spreadsheets/:id` | Удалить |
| POST | `/api/spreadsheets/:id/permissions` | Дать доступ пользователю |
| DELETE | `/api/spreadsheets/:id/permissions/:userId` | Убрать доступ |
| PATCH | `/api/spreadsheets/:id/lock` | Заблокировать/разблокировать |
| PATCH | `/api/spreadsheets/:id/backup-toggle` | (устарело, не используется) |

### WebSocket события

| Событие | Направление | Описание |
|---------|-------------|----------|
| `join-sheet` | client → server | Войти в комнату таблицы |
| `cell-change` | оба | Изменения для real-time синхронизации |
| `save-sheet` | client → server | Сохранить состояние в БД + запись в историю |
| `room-users` | server → client | Список кто сейчас в таблице |
| `changelog-update` | server → client | Новая запись истории (для всех в комнате) |

## Установка на сервер (первый раз)

```bash
# Подключиться к серверу
ssh -i vps/id_rsa root@<YOUR_SERVER_IP>

# Запустить скрипт установки
bash /root/setup-vps.sh
```

После установки: **http://<YOUR_SERVER_IP>**  
Первый вход: `admin` / `admin123` — **сразу сменить пароль!**

---

## Обновление кода на сервере

```bash
# Сначала загрузить изменения в git
git push origin main

# Потом применить на сервере
ssh -i vps/id_rsa root@<YOUR_SERVER_IP> "bash /var/www/tableweb/deploy/update.sh"
```

При обновлении: если добавились новые таблицы в БД — нужно выполнить дополнительно:
```bash
ssh -i vps/id_rsa root@<YOUR_SERVER_IP> "sudo -u postgres psql -d tableweb -f /var/www/tableweb/app/backend/src/db/schema.sql"
```
